Search

Search vendors and binds

Search for vendors and their bindings by token or login. Returns vendors data with their associated bindings information.

Authorizations:
RefererAuthApiKeyAuth
query Parameters
token
required
string
Example: token=4a57e4c3-9877-487b-9a2f-29bae392556e

Token or login to search for vendors Required parameter

Responses

Binder production server

https://bapi88.apitter.com/api/v1/int/search

Request samples

  • cURL Example
curl -X GET "https://bapi88.apitter.com/api/v1/int/search?token=4a57e4c3-9877-487b-9a2f-29bae392556e" \
-H "Accept: application/json" \
-H "X-API-Key: secret" \

Get vendors

Returns a list of vendors. Can be filtered by login or use pagination through limit and offset parameters. Either login or both limit/offset must be provided.

Authorizations:
RefererAuthApiKeyAuth
query Parameters
login
string
Example: login=login777

Filter by vendor login. Required when limit and offset are not provided

limit
integer [ 1 .. 100 ]
Example: limit=10

Maximum number of records to return Required when login is not provided

offset
integer >= 0
Example: offset=0

Number of records to skip for pagination Required when login is not provided

Responses

Binder production server

https://bapi88.apitter.com/api/v1/int/getVendors

Request samples

  • cURL Example
curl -X GET "https://bapi88.apitter.com/api/v1/int/getVendors?limit=10&offset=0" \
-H "Accept: application/json" \
-H "X-API-Key: secret" \
-H "Referer: https://somedomain.com/"

Response samples

Integration

Connect CRM with messaging platform

Previously available by binder call: POST https://b2288.apitter.com/403*****-****-****-****-3be******712?keykey=adminkey&json Initializes and creates integration between a CRM system and a messaging platform. Automatically identifies CRM type from the provided parameters and creates necessary vendor records and bindings.

Authorizations:
RefererAuthApiKeyAuth
Request Body schema: application/json
required
touchapi_source
required
string

Messaging platform source (e.g., whatsapp, telegram)

touchapi_token
required
string

Token for the messaging platform vendor

touchapi_login
required
string

Login for the messaging platform vendor

crm_url
required
string

CRM domain URL. Supported formats:

  • domain.amocrm.ru
  • domain.kommo.com
  • domain.bitrix24.ru
  • domain.bitrix24.com
crm_token
required
string

Authentication token for the CRM system

touchapi_main
boolean

Special flag for AmoCRM integration indicating main instance. When true, clears touchapi_main flag from other instances.

touchapi_name
string

Custom name for the integration (used in AmoCRM source naming)

touchapi_phone
string

Phone number associated with the integration

Responses

Binder production server

https://bapi88.apitter.com/api/v1/int/connect
